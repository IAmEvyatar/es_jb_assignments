Notes on POST

  • Appends form-data inside the body of the HTTP request (data is not shown is in URL)
  • Has no size limitations
  • Form submissions with POST cannot be bookmarked
  • Can be used with HTTPS for secure communications of personal details to server.

Use the Devtools network tab to monitor the post request sent by submitting the form below.

Notes on GET

  • Appends form-data into the URL in name/value pairs
  • The length of a URL is limited (about 3000 characters)
  • Never use GET to send sensitive data! (will be visible in the URL)
  • Useful for form submissions where a user want to bookmark the result
  • GET is better for non-secure data, like query strings in Google
  • A good use for GET would be search engine so users could copy the result url and share a search result with others.
